## Проект 1: Тестирование API расчёта быстрой доставки (Яндекс Прилавок)

## Цель проекта
Проверить корректность работы API расчёта быстрой доставки курьерской службой "Привезём быстро".

## Что было сделано
- Проанализирована документация API (Swagger)
- Составлен функциональный [чек-лист](fast-delivery-checklist.pdf) для проверки:
  - позитивных сценариев 
  - негативных сценариев 
  - структуры и содержимого XML-ответа
- Создана [коллекция в Postman](fast-delivery-postman-collection.json)
- Оформлены [баг-репорты](fast-delivery-bug-report.pdf) по найденным дефектам

## Результаты
В ходе тестирования были выявлены дефекты, связанные с:
- некорректной валидацией входных параметров
- расхождением формата ответа с документацией
- некорректным текстом сообщений об ошибках

Все дефекты оформлены в виде баг-репортов с описанием шагов воспроизведения, ожидаемым и фактическим результатом.

## Выводы
В рамках проекта получила практический опыт:
- анализа API-документации
- составления чек-листа
- работы с HTTP-запросами
- анализа XML-ответов
- оформления баг-репорта

## Использованные инструменты
- Swagger — анализ API-документации
- Postman — создание коллекции, тест-ран, автопроверки
- DevTools — анализ запросов и ответов
